# راه‌اندازی خودکارسازی رایگان (GitHub Actions + Gemini)

این راهنما را یک‌بار، گام‌به‌گام دنبال کن. حدود ۱۵-۲۰ دقیقه زمان می‌برد. بعدش، هر روز
صبح خودش بدون دخالت تو اخبار تازه را می‌گیرد، ترجمه می‌کند، و سایتت را به‌روز می‌کند.

---

## گام ۱ — گرفتن API Key رایگان از Gemini

1. برو به https://aistudio.google.com/app/apikey
2. با حساب گوگلت وارد شو.
3. روی «Create API key» بزن.
4. کلیدی که ساخته می‌شود را کپی کن و یک‌جا نگه دار (بعداً لازمش داری). این کلید رایگان است
   و سهمیه‌ی روزانه‌ی رایگانش برای این پروژه (چند ده خبر در روز) کاملاً کافی است.

## گام ۲ — ساخت ریپازیتوری در GitHub

1. اگر حساب گیت‌هاب نداری، در https://github.com/signup یکی بساز (رایگان).
2. وارد گیت‌هاب شو و روی «New repository» بزن.
3. یک اسم بده، مثلاً `tech-news-fa`. آن را **Public** بگذار (برای GitHub Pages رایگان لازم است).
4. روی «Create repository» بزن.

## گام ۳ — آپلود فایل‌های این پروژه

در صفحه‌ی ریپازیتوری تازه‌ساخته‌شده:
1. روی «uploading an existing file» کلیک کن.
2. تمام فایل‌های زیر را از همین پکیج بکش‌وبیندار (drag & drop):
   - `index.html`
   - `news-dataset.json`
   - `update_news.py`
   - `requirements.txt`
   - `README.md`
   - پوشه‌ی کامل `.github` (شامل `.github/workflows/update-news.yml`)

   ⚠️ نکته: گیت‌هاب گاهی پوشه‌ها را در آپلود وب‌سایتی نمی‌پذیرد. اگر `.github` آپلود نشد:
   - روی «Add file» → «Create new file» بزن.
   - در قسمت نام فایل بنویس: `.github/workflows/update-news.yml`
     (با تایپ `/` گیت‌هاب خودش پوشه می‌سازد.)
   - محتوای فایل `update-news.yml` را در آن کپی کن.
3. پایین صفحه روی «Commit changes» بزن.

## گام ۴ — اضافه‌کردن کلید Gemini به‌صورت امن (Secret)

1. در صفحه‌ی ریپازیتوری برو به: **Settings** → **Secrets and variables** → **Actions**
2. روی «New repository secret» بزن.
3. در «Name» بنویس: `GEMINI_API_KEY`
4. در «Secret» همان کلیدی که در گام ۱ کپی کردی را بگذار.
5. «Add secret» را بزن.

## گام ۵ — روشن‌کردن GitHub Pages (برای دیدن سایت آنلاین)

1. برو به **Settings** → **Pages**.
2. زیر «Build and deployment» → «Source» را روی **Deploy from a branch** بگذار.
3. شاخه (Branch) را `main` و پوشه را `/ (root)` انتخاب کن، و «Save» بزن.
4. بعد از چند دقیقه، آدرس سایتت در همان صفحه نمایش داده می‌شود؛ چیزی شبیه:
   `https://USERNAME.github.io/tech-news-fa/`

## گام ۶ — تست دستی (پیش از منتظر ماندن تا فردا)

1. برو به تب **Actions** در ریپازیتوری.
2. روی workflow با اسم «به‌روزرسانی روزانه‌ی آرشیو اخبار» کلیک کن.
3. سمت راست روی «Run workflow» بزن تا همین الان یک‌بار دستی اجرا شود.
4. بعد از حدود یک دقیقه، صفحه را رفرش کن؛ اگر علامت سبز ✅ دیدی، یعنی کار کرد و
   `news-dataset.json` به‌صورت خودکار آپدیت و commit شد.
5. سایتت (آدرس گام ۵) را باز کن — باید خبرهای تازه را ببینی.

---

## از این به بعد چه اتفاقی می‌افتد؟

هر روز ساعت ۶ صبح (به وقت ایران)، گیت‌هاب خودش:
1. فید سه‌تا سایت (OpenAI، TechCrunch، MIT Tech Review) را می‌خواند.
2. خبرهایی که قبلاً ثبت نشده‌اند را پیدا می‌کند.
3. هرکدام را با Gemini ترجمه و خلاصه می‌کند.
4. به `news-dataset.json` اضافه می‌کند و خودش commit/push می‌کند.
5. سایتت (GitHub Pages) خودش با همین فایل تازه به‌روز می‌شود — کاری از طرف تو لازم نیست.

## اگر چیزی کار نکرد

- تب **Actions** را چک کن؛ روی اجرای ناموفق (علامت ضربدر قرمز ❌) کلیک کن تا پیام خطا را ببینی.
- خطای رایج: کلید Gemini اشتباه کپی شده یا هنوز فعال نشده — دوباره از گام ۱ چک کن.
- اگر «هیچ خبر تازه‌ای پیدا نشد» دید، طبیعی است — یعنی از دفعه‌ی قبل خبر جدیدی منتشر نشده.

## نکته‌ی هزینه

این مسیر کاملاً رایگان است: GitHub Actions برای ریپازیتوری‌های Public رایگان است،
GitHub Pages رایگان است، و Gemini API در سهمیه‌ی رایگان روزانه‌اش (که برای حجم خبر روزانه
بیش از کافی است) هزینه‌ای ندارد.
