# رصدخانه — آرشیو اخبار تکنولوژی و هوش مصنوعی به فارسی

## فایل‌های این پکیج
- **index.html** — صفحه‌ی اصلی سایت. داده‌ها را از `news-dataset.json` کنار خودش می‌خواند (دیگر جاسازی‌شده نیست، چون باید با اسکریپت خودکار به‌روز شود).
- **news-dataset.json** — دیتاست خام و ساختاریافته‌ی همه‌ی اخبار. پایه‌ی هر تحلیل آینده.
- **update_news.py** — اسکریپت پایتون که فیدها را می‌خواند، با Gemini ترجمه می‌کند، و دیتاست را به‌روز می‌کند.
- **.github/workflows/update-news.yml** — فایل GitHub Actions که هر روز خودش `update_news.py` را اجرا می‌کند.
- **requirements.txt** — نیازمندی‌های پایتون.
- **SETUP.md** — راهنمای گام‌به‌گام راه‌اندازی خودکارسازی رایگان (GitHub + Gemini).

## دو مسیر استفاده

### مسیر ۱: فقط مشاهده‌ی محلی (بدون خودکارسازی)
کافی است `index.html` و `news-dataset.json` را کنار هم در یک پوشه نگه داری و
`index.html` را در مرورگر باز کنی. هر وقت خواستی به‌روزش کنی، در یک مکالمه با Claude
بگو: «اخبار جدید رو بگیر»، فایل `news-dataset.json` تازه را جایگزین کن.

⚠️ نکته: چون حالا `index.html` با `fetch()` فایل JSON را می‌خواند، اگر به‌جای باز
کردن مستقیم فایل (`file://`) با خطای CORS مواجه شدی، یک سرور محلی ساده اجرا کن:
```
python3 -m http.server 8000
```
و در مرورگر به آدرس `http://localhost:8000` برو.

### مسیر ۲: خودکارسازی کامل و رایگان (پیشنهادی)
فایل **SETUP.md** را باز کن و گام‌به‌گام دنبالش کن. بعد از یک راه‌اندازی یک‌باره
(حدود ۱۵ دقیقه)، سایتت هر روز خودش بدون دخالت تو به‌روز می‌شود — رایگان، با
GitHub Actions و Gemini API.

## ساختار هر رکورد خبر (برای تحلیل آینده)
```json
{
  "id": "شناسه‌ی یکتا (هش لینک خبر)",
  "date": "تاریخ میلادی انتشار (YYYY-MM-DD)",
  "source": "نام منبع",
  "source_url": "لینک خبر اصلی",
  "category": "دسته‌بندی موضوعی",
  "title_fa": "عنوان فارسی",
  "title_en": "عنوان اصلی انگلیسی",
  "summary_fa": "خلاصه‌ی روان فارسی",
  "tags": ["برچسب‌های کلیدی"]
}
```
بعد از جمع‌آوری چند ماه/سال داده، می‌توان این فایل JSON را به یک ابزار تحلیل
(پایتون/پانداس، یا خود Claude) داد و روند تغییرات صنعت هوش مصنوعی را بر اساس
`category`، `tags` و `date` بررسی کرد.
